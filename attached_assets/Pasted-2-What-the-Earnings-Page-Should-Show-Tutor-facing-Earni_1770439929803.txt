3. Notifications Behaviour (Very Important UX Fix)

You’re also right here. A notification should only be “new” once.

Expected Behaviour

Notification badge count reflects unread notifications only

Once a user opens the notifications panel:

All currently visible notifications are marked as read

The badge count clears immediately

Badge should only reappear when:

A brand new notification is created after that moment

What Should NOT Happen

Old notifications reappearing as “new”

Badge persisting after user has viewed notifications

Badge resetting on page refresh

4. Exact Instructions for the Agent / Developer

You can give them this verbatim.

A. Earnings UI Changes

Remove the “Platform Fee (25%)” card from the tutor earnings dashboard.

Display only:

Gross Earnings

Available for Withdrawal

Ensure gross earnings are not visually paired with negative or red indicators.

Add a small info tooltip or footnote text near Gross Earnings:

“Gross earnings are subject to a standard platform management fee as outlined in the Tutor Terms.”

B. Fee Handling Logic

Keep the 25% platform fee logic unchanged in backend calculations.

Apply the fee:

At withdrawal request

In payout history records

Do not surface fee deductions in the main dashboard summary.

C. Notifications Logic

Track notification state using a read / unread flag per notification.

When a tutor opens the notifications panel:

Mark all currently unread notifications as read

Immediately clear the notification badge count.

Only increment the badge when:

A new notification is created after the last read timestamp.

Do not re-mark notifications as unread on refresh, navigation, or login.
