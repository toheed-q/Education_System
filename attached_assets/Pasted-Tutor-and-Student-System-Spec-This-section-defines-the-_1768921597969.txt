Tutor and Student System Spec

This section defines the super categories, dashboards, and registration journeys explicitly. Build exactly as described. Do not infer missing steps. Do not replace with “etc.”

1) Super categories explained explicitly
1.1 Why super categories exist

Super categories exist to:

Prevent tutors from appearing in school child categories without verification

Make it impossible to bypass child safety rules by choosing unrelated subjects

Ensure search filters map a parent’s selection (for example Grade 4) to the correct tutor pool

Allow verification and visibility decisions per super category

Each tutor may select one or more super categories. Each super category has its own subject list and its own verification status. The system must treat each super category as a separate compliance scope.

1.2 The three super categories (fixed list)

Super Category A: School Tutoring

Includes all tutoring for learners in:

Pre-Primary (PP1, PP2)

Lower Primary (Grade 1–3)

Upper Primary (Grade 4–6)

Junior Secondary (Grade 7–9)

Senior Secondary (Grade 10–12)

This category involves minors. It is high-risk. It requires strict verification before tutors can appear publicly in this category.

Verification rules for School Tutoring

Tutor must pay KES 500 one-time verification fee.

Tutor must submit required documents:

National ID

TSC certificate and or TSC number

Admin must approve School Tutoring verification before the tutor is visible for School Tutoring search results or bookings.

Super Category B: Higher Education

Includes tutoring for post-secondary learners:

Certificate

Diploma

Undergraduate

Postgraduate

This category does not require TSC. Verification is optional but supported to increase trust.

Verification rules for Higher Education

Tutor pays KES 300 one-time verification fee, which covers both Higher Education and Professional Skills as a combined verification fee.

Required documents for approval:

National ID

Degree or professional certificate

Admin approves verification for Higher Education separately.

Super Category C: Professional Skills

Includes workplace skills tutoring and mentorship:

Business mentoring

Career coaching

Product management coaching

Professional writing

Public speaking

Data analytics coaching

Verification is optional but supported to increase trust.

Verification rules for Professional Skills

Tutor pays KES 300 one-time combined verification fee (covers Higher Education and Professional Skills together).

Required documents for approval:

National ID

Professional certificate or credible proof

Admin approves verification for Professional Skills separately.

1.3 Fee rule summary (must implement)

If tutor selects School Tutoring in any way:

They must pay KES 500 to be eligible for School Tutoring verification.

If tutor selects only Higher Education and or Professional Skills:

They must pay KES 300 for verification eligibility.

Payment is one-time per tutor per fee type. It is not per subject.

Admin verification is per super category, regardless of fee.

The system must store:

what fee was paid (KES 500 or KES 300)

what categories were paid for eligibility

what categories were actually approved

1.4 Visibility rules tied to super categories

Visibility is not one global switch. Visibility is per category.

Example

Tutor selects School Tutoring and Professional Skills.

Tutor has not been verified for School Tutoring yet.

Tutor may be visible in Professional Skills listings but must be hidden from School Tutoring listings until approved.

On the tutor profile page shown to students/parents

Show the three super categories with status badges:

Approved: check mark and “Verified”

Pending: “Verification in progress”

Not verified: “Not verified”

If School Tutoring is not verified, show a clear statement:

“This tutor is not verified for School Tutoring and cannot be booked for school learners.”

3) Tutor registration journey (all angles, fully explicit)
3.1 Tutor onboarding stages

Tutor onboarding must have clear stages:

Stage 1: Account creation

Tutor enters:

full name

email

phone number

password

System sends OTP to email

Tutor enters OTP

Account becomes active

Role is set to TUTOR

Stage 2: Profile setup wizard
Wizard pages:

Profile photo upload (required)

Bio (required)

Choose super categories (choose 1 to 3)

Choose subjects under each selected super category (required)

Set hourly rate (KES) (required)

Choose online and or in-person offering

If in-person selected:

location required

Set availability schedule (required)

Review and submit profile

Profile completion percent

Must compute strictly from required fields

Profile becomes eligible to list only at 100%

Stage 3: Verification setup

Tutor sees verification panel for each selected super category

Each category shows:

status

required docs

required fee for eligibility

Verification fee logic screens

If School Tutoring selected:

show “School Tutoring verification fee: KES 500 one-time”

If only Higher Education and or Professional Skills selected:

show “Verification fee: KES 300 one-time (covers Higher Education and Professional Skills)”

Stage 4: Document upload

Tutor uploads docs based on selected categories

Docs stored privately and accessible only via signed URLs

Stage 5: Pay and submit for verification

Tutor clicks “Submit for verification”

System checks:

profile completion must be 100

required docs present for the selected category

If checks pass:

system sets status to AWAITING_PAYMENT

initiates Paystack payment

Only after webhook confirms:

status becomes IN_REVIEW

admin can see request

Stage 6: Admin review

Admin approves or returns for resubmission or rejects per category

Tutor sees reason if returned or rejected

Tutor receives email updates

Stage 7: Listing rules

Tutor becomes visible:

in categories that are either:

verified approved, or

do not require verification

Tutor can be visible in Professional Skills while School Tutoring remains pending

3.2 Tutor registration edge cases

Tutor registers but never completes profile:

keep account but set is_visible false

Tutor completes profile but never pays verification:

visible only in non-verified categories

Tutor selects School Tutoring later after being active in Professional Skills:

immediately hide them from School Tutoring listings until School Tutoring verification approved

Tutor resubmits docs:

no new payment required

Tutor changes phone or email:

must re-verify OTP

4.2 Course enrollment journey from all angles

Case 1: Logged-in student

student clicks Enroll

system calls backend payment init for COURSE

Paystack payment completes

backend webhook confirms and fulfills

“Payment confirmed”

Case 2: Logged-out visitor

visitor clicks Enroll

collect:

full name

email

phone

payment init and Paystack payment

backend confirms via webhook

create or link account:

if email already exists, prompt login

if phone already exists with different email, show error and ask to contact support

after account link:

show “Set password” screen or offer Google login

then redirect to course dashboard
